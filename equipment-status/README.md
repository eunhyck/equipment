장비 상태 관리 시스템

(Equipment Status Management System)

1. 프로젝트 개요

본 프로젝트는 조직에서 사용하는 다양한 IT/사무 장비의 현황을 등록·조회·수정·삭제하고,
장비의 **상태(정상 / 점검중 / 장애)**를 직관적으로 관리하기 위한 웹 기반 관리 시스템입니다.

장비 등록과 조회를 분리하고, 상세수정 페이지를 별도 구성하여
운영자가 실제 현장에서 바로 사용할 수 있는 형태의 실무형 CRUD 시스템으로 설계·구현했습니다.

2. 기획 배경

운영 환경에서는 다음과 같은 문제가 자주 발생합니다.

장비 정보가 엑셀, 메신저 등 여러 곳에 흩어져 있음

현재 상태(정상/장애/점검)를 한눈에 파악하기 어려움

변경 이력과 최종 수정 시각을 관리하기 어려움

이를 해결하기 위해 본 프로젝트는 다음을 목표로 합니다.

장비 정보를 일원화하여 관리

웹 화면에서 간편하게 CRUD 수행

상태정보와 마지막 변경 일시를 함께 관리

상세수정 전용 화면 제공으로 실수 방지

3. 주요 기능

1) 장비 등록

장비명, 담당자, 위치, 상태 입력 후 저장

기본 상태는 “정상”

등록 후 목록 화면에서 즉시 확인 가능

2. 장비 조회

전체 장비 목록 테이블 제공

상태를 색상/배지로 시각화 표시

최종 변경 일시 확인 가능

3. 장비 상세 수정

목록에서 상세수정 버튼 클릭 시 수정 화면으로 이동

개별 장비의 모든 항목 수정 가능

수정 시 수정시간 자동 업데이트

4. 장비 삭제

목록 화면에서 삭제 버튼 제공

삭제 전 확인 알림 표시

4. 시스템 구조
   Frontend : HTML + CSS + JavaScript
   Backend : Node.js(Express)
   Database : Oracle DB
   API방식 : RESTful API

핵심 구조

/api/equipments REST API 제공

등록(POST) / 조회(GET) / 수정(PUT) / 삭제(DELETE)

상태만 수정하는 전용 API도 분리 설계 가능

프론트엔드와 백엔드 완전 분리형 구조

5. 기술 스택

6. 실행 방법

1) 패키지 설치
   npm install

2) Oracle DB 테이블 생성
   CREATE TABLE EQUIPMENT (
   EQUIP_ID NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
   EQUIP_NAME VARCHAR2(100) NOT NULL,
   MANAGER VARCHAR2(50),
   STATUS VARCHAR2(20) NOT NULL,
   LOCATION VARCHAR2(100),
   UPDATED_AT DATE DEFAULT SYSDATE
   );

3) 서버 실행
   node server.js

4) 접속
   http://localhost:3000
